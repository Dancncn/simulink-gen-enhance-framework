# ROADMAP

## P0（已完成 / 持续优化）
目标：打通“生成 + 仿真 + 报告”闭环。

范围：
- 由 `gen_model.m` 生成最小可仿真模型
- 由 `matlab/run_verify.m` 完成阶段化验证与 JSON 报告
- 由 `verify.py` 提供跨 shell 的稳定入口
- 运行产物统一落地 `runs\<timestamp>\`

## P1（计划，明天执行）
目标：从“能跑通”升级为“结构与指标合规”。

计划能力：
- 结构判题（必须块/禁止块）
- 连接拓扑检查
- 时域/频域指标门槛
- 失败原因结构化输出

### 明天要做的 P1 任务清单（<=10）
1. 在 `run_verify` 中增加结构检查入口（独立函数）。
2. 定义禁止块清单（如 `Transfer Fcn`、`State-Space`）。
3. 定义必须块清单（至少一条积分链）。
4. 增加模型拓扑检查（关键链路完整性）。
5. 增加时域指标计算（超调、稳态误差、调节时间）。
6. 增加频域指标检查（例如 PSD 或带宽约束）。
7. 将结构与指标结果写入 `verify_report.json.metrics`。
8. 统一错误码与 `fail_stage` 语义，便于 agent 自动修复。
9. 增加 2~3 个固定测试 case（合格/不合格/边界）。
10. 更新 README 与 FAQ 的 P1 使用说明。
